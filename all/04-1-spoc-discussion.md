### 8.1 虚拟存储的需求背景

1. 寄存器、高速缓存、内存、外存的访问特征？

   访问速度递减，容量递增

2. 如何理解计算机中的存储层次结构所的理想状态是“均衡繁忙”状态？

   这种理想状态是指各个层次的存储介质都处于繁忙的状态，但是又不构成系统瓶颈

3. 在你写程序时遇到过内存不够的情况吗？尝试过什么解决方法？

   有，例如在训练模型时，无法将所有的训练数据都存储到内存中，但是不读入就会使得读取数据成为跑算法的瓶颈。解决方法时，设置一个队列，队列大小适当，读取数据和训练多进程进行。

### 8.2 覆盖和交换

1. 什么是覆盖技术？使用覆盖技术的程序开发者的主要工作是什么？

   覆盖技术是指，当一个程序可以按照逻辑分成若干个没有调用关系的组时，通过相同组模块之间的覆盖、替换来节省内存空间。

   程序员需要进行模块划分，确定模块之间的调用关系。

2. 什么是交换技术？覆盖与交换有什么不同？

   交换技术是指在进程之间的内存交换，当一个进程需要更大的内存空间时，但是已经没有足够的空闲内存空间，此时将部分进程存储到外存中，来空出足够的内存。

   覆盖是指在一个进程之内的内存替换、而交换是在进程之间的内存交换。

3. 覆盖和交换技术在现代计算机系统中还有需要吗？可能用在什么地方？

   

4. 如何分析内核模块间的依赖关系？

   使用depmod命令

5. 如何获取内核模块间的函数调用列表？

   静态代码分析，动态跟踪程序执行


### 8.3 局部性原理

1. 什么是时间局部性、空间局部性和分支局部性？

   时间局部性：如果在某个时间访问了某个存储器中某个位置，很有可能在不久再次访问同一个位置。

   空间局部性：在某个时间访问了存储中某个数据，很有可能在不久访问相邻的位置

   分支局部性：跳转指令在相邻的时间内，很有可能跳转到同一个地址

2. 如何提高程序执行时的局部性特征？

   有意识的规范数据访问次序和程序跳转

3. 排序算法的局部性特征？
  * 参考：[九大排序算法再总结](http://blog.csdn.net/xiazdong/article/details/8462393)

### 8.4 虚拟存储概念

1. 什么是虚拟存储？它与覆盖和交换的区别是什么？它有什么好处和挑战？

   虚拟存储：只把当前指令执行所需要的进程空间，加载到物理内存中

   特征：不连续性、大用户空间、部分交换

2. 虚拟存储需要什么样的支持技术？

   需要硬件上的地址转换和操作系统上的置换算法


### 8.5 虚拟页式存储

 1. 什么是虚拟页式存储？缺页中断处理的功能是什么？

    虚拟页式存储是指开始只将程序的部分页加载到内存中，并之后根据进程需要动态加载页面。当内存使用已满时，还可以将部分页面换出到磁盘上。缺页中断处理可以帮助加载不在内存中的页面。

 1. 为了支持虚拟页式存储的实现，页表项有什么修改？

    需要增加多个标志位

    驻留位：表示是否在物理内存中

    修改位：是否被修改过，如果被修改过，需要在替换时写到物理内存中。

    访问位：是否被访问过

    保护位：表示访问权限(只读、可写等)


 2. 页式存储和虚拟页式存储的区别是什么？

    虚拟页式存储相对于页式存储多了需要判断是否缺页等操作，即虚拟页式存储中需要的页并不全部加载到内存中。

### 8.6 缺页异常

1. 缺页异常的处理流程？

   1） 指令执行，查找页表项

   2） 页表项驻留位为0，触发缺页异常

   3） 在外存中查找对应页

   4） 页面换入

   5） 页表项修改

   6） 重新执行指令


2. 虚拟页式存储管理中有效存储访问时间是如何计算的？

   内存访问时间 * 命中率 + 缺页率 * 缺页异常处理时间

### 虚拟页式存储的地址转换

（3）(spoc) 有一台假想的计算机，页大小（page size）为32 Bytes，支持8KB的虚拟地址空间（virtual address space）,有4KB的物理内存空间（physical memory），采用二级页表，一个页目录项（page directory entry ，PDE）大小为1 Byte,一个页表项（page-table entries
PTEs）大小为1 Byte，1个页目录表大小为32 Bytes，1个页表大小为32 Bytes。页目录基址寄存器（page directory base register，PDBR）保存了页目录表的物理地址（按页对齐）。

PTE格式（8 bit） :
```
  VALID | PFN6 ... PFN0
```
PDE格式（8 bit） :
```
  VALID | PT6 ... PT0
```
其
```
VALID==1表示，表示映射存在；VALID==0表示，表示内存映射不存在（有两种情况：a.对应的物理页帧swap out在硬盘上；b.既没有在内存中，页没有在硬盘上，这时页帧号为0x7F）。
PFN6..0:页帧号或外存中的后备页号
PT6..0:页表的物理基址>>5
```

已经建立好了1个页目录表和8个页表，且页目录表的index为0~7的页目录项分别对应了这8个页表。
